# Habits — career

> Syntax rules (carryover logic depends on exact format):
> Open: `- [ ] item` · Done: `- [x] item` · Dropped: `- [x] ~~item~~ (dropped: reason)`

## Daily

- [ ] Block the first 3–4 hours of the workday as protected deep-work time (no meetings, no Slack/email)
- [ ] Identify the single most important task for the day before opening any communications
- [ ] Log deep-work hours at end of workday (app or manual note)
- [ ] End the day with a 5-min shutdown ritual: note tomorrow's top task, close all tabs <!-- TODO: personalize -->

## Weekly

- [ ] Review deep-work hours for the week — compare to 20 h target
- [ ] Review W3Mirror portfolio status at a high level (revenue, key signals) — read from control-pane
- [ ] Identify one skill-practice session this week (deliberate practice on the current skill roadmap item)
- [ ] Make at least one meaningful professional connection (reply, comment, message, or call)
- [ ] Review `career/ROADMAP.md` — any confirmed items to move forward this week?

## Periodic

- [ ] Publish one public artefact per month (article, thread, open-source PR, or talk)
- [ ] Update skill roadmap progress in `resources/skills/roadmap-2026.md` (monthly, first week)
- [ ] Quarterly career review: goals progress, skills acquired, income vs. target, roadmap update
- [ ] Annual career review: 12-month look-back, next-12-month plan, update north-star if needed
